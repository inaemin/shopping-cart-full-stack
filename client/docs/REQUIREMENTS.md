## 요구사항 정리

- [ ] 장바구니 상품 데이터를 렌더링한다.
  - 장바구니 GET api를 사용한다.
  - 상품의 선택 여부, 결제 금액, 배송비 등의 상태를 관리한다.
  - 상품의 선택 여부는 새로고침 시에도 유지되어야 한다.
  - 상품은 기본적으로 선택 상태이다.
  - 선택/해제 시 결제 금액을 동적으로 변경한다.
  - 결제 금액이 10만원 이상의 경우 배송비는 무료이다.
- [ ] 장바구니 상품의 수량을 변경할 수 있다.
  - 장바구니 PATCH api를 사용한다.
- [ ] 장바구니 상품을 제거할 수 있다.
  - 장바구니 DELETE api를 사용한다.
- [ ] 비동기 상태에 따라 적절한 UI를 보여준다.
  - 로딩 상태 -> 스켈레톤, 스피너..
  - 에러 -> 에러메시지를 활용한 UI 표시.

## ✅ 프로그래밍 요구사항

> 이전 미션의 프로그래밍 요구사항은 기본으로 포함한다.

### State Management

- React의 상태 관리 훅(useState, useReducer)을 사용하여 상태를 구성하고, 필요 시에 Context API 등을 활용해 해당 상태가 필요한 컴포넌트 트리에 상태를 전달합니다.
- 결제 금액, 배송비 등의 파생 상태(Derived State)를 동적으로 계산한다.

### 코드 퀄리티

- 컴포넌트와 비즈니스 로직을 분리하여, 재사용 가능하게 한다.
  - 재사용 가능하게 하기 위해서, 컴포넌트 내에서 함수를 호출하는 것과, props 를 가지는 방식을 구별하고, 더 나은 방법을 채택한다.
- 사용하기 좋은 인터페이스를 고민한다. 이 컴포넌트의 다양한 use case 를 떠올려본다.
- refetchCart, deleteCartItem 같이 세부 구현을 나타내는 props 가 있다면 이는 이름부터 안티 패턴이다. 다른 use case 에 대응이 어렵다. 1단계 서버 미션에서 다룬, DIP 를 떠올려보자.
